DocForge – Agentic Documentation Orchestration
Phase 1 MVP: Change-Driven, Explainable Documentation Proposals

DocForge MVP validates whether an agentic system can detect real engineering changes, infer documentation impact, propose precise updates, and explain reasoning transparently enough for human trust/rejection — all without becoming authoritative, adding process overhead, or creating data gravity.

Core MVP Technologies & Constraints

  • • Python / TypeScript + open-source LLM (local-first)
  • • Git CLI for change detection & proposals
  • • Minimal CLI validation (no dashboards, no auth)
  • • Structured JSON traces stored in-repo
  • • Strict constraints: no authority, no memory, no cross-repo

MVP Architectural Goal & What It Proves

Single Goal: Prove an agent can infer doc impact from real changes, propose updates, and earn trust through explainability — not automation.

What the MVP Proves

  • Agentic reasoning > deterministic pipelines for doc synthesis
  • Impact inferable from artifacts
  • Trust via explainability
  • Viable human-in-loop without heavy UI

What It Does NOT Prove

  • Scalability
  • Completeness
  • Autonomous correctness
  • Platform viability

MVP Thin Vertical Slice Flow

Component Breakdown

  • Signal Ingestor: Dumb classification of git/ADR changes (critical separation for auditability)
  • Reasoning Agent: Forms impact hypotheses with confidence & rationale
  • Validation Layer: CLI human review — architectural requirement, not UX limitation
  • Doc Proposal Engine: Generates patches/PRs annotated with traces
  • Trace Store: In-repo JSON + Markdown for full explainability

Designed Failure Modes & Canonical Demo

Failure Modes (Explicitly Designed)

  • Low confidence → Ask human, do nothing
  • Conflicting signals → Surface ambiguity
  • Agent error → Human rejection
  • Outage → No doc changes (silent failure forbidden)

Canonical Demo Scenario

Modify ADR → Commit → Run DocForge → Agent explains impact → Human validates → Proposal + trace generated. If this loop works, MVP succeeds.

DocForge MVP is deliberately narrow, conservative, and reversible — a thoughtful probe into whether agentic reasoning can meaningfully reduce documentation debt without introducing new architectural problems.
