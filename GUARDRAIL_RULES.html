Guardrail Rules Engine

DocForge includes multiple guardrail layers to ensure generated outputs are accurate, appropriate, and strictly grounded in uploaded specifications. These rules prevent hallucinations, off-topic responses, and style deviations.

Guardrail Layers

Layer Purpose Mechanism Response When Triggered
Document Scope Guard Prevents answers outside uploaded specs Pre-generation check for external knowledge requests "I can only answer based on uploaded specifications"
Minimum Standard Validation Ensures sufficient input quality/quantity Configurable thresholds (e.g., required sections present) Grays out unavailable deliverables with explanation
Style Guide Enforcement 100% adherence to uploaded style guide Post-generation auto-correction agent Automatically fixes grammar, terminology, structure
Hallucination Detection Catches ungrounded claims Self-checking prompt + confidence scoring Re-generates or refuses with explanation
Citation Enforcement Ensures traceability Post-generation validation Forces inclusion of source references

Style Guide Enforcement Flow

  1. User uploads style guide (one-time, changeable)
  2. Agent indexes style rules (grammar, terminology, structure)
  3. After generation, StyleEnforcer agent reviews output
  4. Auto-corrects deviations (no human in loop)
  5. Writer receives 100% compliant draft for final refinement
